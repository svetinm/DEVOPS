Ovo bih radio kao **LO4 + LO5 bootcamp**. Cilj nije naučiti napamet YAML nego da za svaki problem znaš što upisati.

# LAB 1 – Deployment i Service

## 1. Kreiraj deployment

```bash
kubectl create deployment web \
  --image=nginx:1.25
```

Provjera:

```bash
kubectl get deployments
kubectl get rs
kubectl get pods
```

Pitanja:

* Što je Deployment?
* Što je ReplicaSet?
* Što je Pod?

---

## 2. Skaliranje

```bash
kubectl scale deployment web \
  --replicas=3
```

Provjera:

```bash
kubectl get pods
kubectl get rs
```

---

## 3. Izloži aplikaciju

```bash
kubectl expose deployment web \
  --port=80 \
  --type=NodePort
```

Provjera:

```bash
kubectl get svc
```

---

## 4. Test

```bash
minikube service web --url
```

Otvori URL.

---

# LAB 2 – Rollout i Rollback

## Nadogradnja

```bash
kubectl set image deployment/web \
  nginx=nginx:1.27
```

Provjera:

```bash
kubectl rollout status deployment/web
```

---

## Povijest

```bash
kubectl rollout history deployment/web
```

---

## Povratak

```bash
kubectl rollout undo deployment/web
```

Provjera:

```bash
kubectl describe deployment web
```

# LAB 3 – Export YAML

Kreiraj:

```bash
kubectl create deployment apache \
  --image=httpd:2.4 \
  --dry-run=client \
  -o yaml > apache.yaml
```

Pregled:

```bash
cat apache.yaml
```

Primijeni:

```bash
kubectl apply -f apache.yaml
```

Briši:

```bash
kubectl delete -f apache.yaml
```

# LAB 6 – PersistentVolumeClaim

Provjeri:

```bash

```

---

PVC.yaml

```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: test-pvc
spec:
  accessModes:
  - ReadWriteOnce
  resources:
    requests:
      storage: 1Gi
```

Primjena:

```bash
kubectl apply -f pvc.yaml
```

Provjera:


# LAB 7 – Job

job.yaml

```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: test-job
spec:
  template:
    spec:
      containers:
      - name: test
        image: busybox
        command: ["echo","Hello"]
      restartPolicy: Never
```

Pokreni:

```bash
kubectl apply -f job.yaml
```

Provjera:

```bash
kubectl get jobs
kubectl get pods
```

Logovi:

```bash
kubectl logs PODNAME
```

---

# LAB 8 – CronJob

cronjob.yaml

```yaml
apiVersion: batch/v1
kind: CronJob
metadata:
  name: test-cron
spec:
  schedule: "* * * * *"
  jobTemplate:
    spec:
      template:
        spec:
          containers:
          - name: date
            image: busybox
            command:
            - date
          restartPolicy: Never
```

Primjena:

```bash
kubectl apply -f cronjob.yaml
```

Provjera:

```bash
kubectl get cronjobs
kubectl get jobs
```

# LO5 – Troubleshooting

Ovo radi svaki dan.

# Problem 1

Kreiraj namjerno loš image:

```yaml
image: nginx:99999
```

Primijeni.

Provjera:

```bash
kubectl get pods
```

Vidjet ćeš:

```text
ImagePullBackOff
```

Dijagnostika:

```bash
kubectl describe pod POD
```

Nađi razlog.

---

# Problem 2

CrashLoopBackOff

Pod:

```yaml
command:
- exit
```

Provjera:

```bash
kubectl get pods
```

Dijagnostika:

```bash
kubectl logs POD --previous
```

---

# Problem 3

Service ne radi

Deployment:

```yaml

  app: nginx
```

```yaml
selector:
  app: apache
```

Provjera:

```bash
kubectl get endpoints
```

Bit će:

```text
<none>
```

Pronađi problem.
